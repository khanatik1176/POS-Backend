# Render Deployment (Django + Neon + Telegram)

This project is now ready to deploy on Render using the included `render.yaml`.

## 1. Create a new Neon production database

1. In Neon, create a separate production database (do not reuse local/dev DB).
2. Copy the connection string.
3. Confirm it includes `sslmode=require`.

Example:

```env
DATABASE_URL=postgresql://<user>:<password>@<neon-host>/<dbname>?sslmode=require
```

## 2. Deploy on Render

1. Push this repo to GitHub/GitLab.
2. In Render, choose **New +** -> **Blueprint**.
3. Connect your repo and deploy. Render will read `render.yaml`.

Included behavior in `render.yaml`:

- Installs dependencies
- Runs `collectstatic`
- Runs `migrate` on startup
- Starts Gunicorn server

## 3. Set required Render environment variables

In Render Dashboard -> Service -> Environment, set:

```env
DEBUG=False
DATABASE_URL=<your-neon-production-url>
CORS_ALLOWED_ORIGINS=https://your-frontend-domain.com
CSRF_TRUSTED_ORIGINS=https://your-frontend-domain.com,https://<your-render-service>.onrender.com
```

Notes:

- `DJANGO_SECRET_KEY` is auto-generated by `render.yaml`.
- `RENDER_EXTERNAL_HOSTNAME` is injected by Render automatically and accepted by settings.

## 4. Telegram configuration from env

Put Telegram values in Render Environment (same as `.env` keys):

```env
TELEGRAM_BOT_ENABLED=True
TELEGRAM_BOT_TOKEN=<bot-token>
TELEGRAM_CHAT_ID=<chat-or-group-id>
TELEGRAM_WEBHOOK_SECRET=<long-random-secret>
TELEGRAM_WEBHOOK_URL=https://<your-render-service>.onrender.com/api/telegram/webhook/
```

After deployment, set webhook once:

```bash
curl -X POST "https://api.telegram.org/bot$TELEGRAM_BOT_TOKEN/setWebhook" \
  -d "url=$TELEGRAM_WEBHOOK_URL" \
  -d "secret_token=$TELEGRAM_WEBHOOK_SECRET"
```

Verify:

```bash
curl "https://api.telegram.org/bot$TELEGRAM_BOT_TOKEN/getWebhookInfo"
```

## 5. Keep local and production DB separated

- Local dev can continue using your local `.env` and local database.
- Render production should use only `DATABASE_URL` pointing to Neon production DB.

## 6. Optional post-deploy commands

If needed, run one-off command in Render shell:

```bash
python manage.py seed_data
```

## 7. Quick checklist

- [ ] Neon production DB created and added to `DATABASE_URL`
- [ ] Render service deployed from `render.yaml`
- [ ] CORS/CSRF origins set for your frontend domain
- [ ] Telegram env vars added in Render
- [ ] Telegram webhook configured and verified
